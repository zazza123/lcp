Getting Started

Generate your first LCP manifest in under 5 minutes.

1 Installation

Install the LCP Python SDK from PyPI:

bash 
pip install lcp

This installs both the Python library and the lcp CLI tool.

2 Your First Scan

Scan any installed Python package to generate its LCP manifest:

bash 
# Scan the requests library
lcp scan requests -o requests.lcp.json

# Scan with private symbols included
lcp scan mypackage --include-private -o mypackage.lcp.json

# Skip validation (faster for large packages)
lcp scan numpy --no-validate -o numpy.lcp.json

The output is a JSON file containing the complete API surface of the library.

3 Understanding the Output

An LCP file has two main sections:

manifest

Metadata about the library: name, version, language, and generation info.

{
  "manifest": {
    "schema_version": "1.0",
    "library": {
      "name": "requests",
      "version": "2.31.0",
      "language": "python"
    },
    "distribution": "pypi"
  }
}

symbols

All public symbols: functions, classes, methods, constants, and modules.

{
  "symbols": {
    "requests:get": {
      "kind": "function",
      "semantics": {
        "summary": "Sends a GET request."
      },
      "signatures": [{
        "params": [
          {"name": "url", "type": "string", "required": true},
          {"name": "timeout", "type": "float", "required": false}
        ],
        "returns": "Response"
      }]
    }
  }
}

4 Using the Python API

You can also use the SDK programmatically:

python 
from lcp import scan

# Scan a package
lcp_doc = scan("requests")

# Save to file
lcp_doc.to_file("requests.lcp.json")

# Get as dictionary
data = lcp_doc.to_dict()

# Access symbols programmatically
for symbol_id, symbol in lcp_doc.symbols.items():
    print(f"{symbol_id}: {symbol.kind}")

# Include private symbols (prefixed with _)
lcp_doc = scan("mypackage", include_private=True)

6 Validating LCP Files

Validate any LCP file against the official schema:

bash 
lcp validate mypackage.lcp.json

This ensures your LCP file conforms to the v1 specification and can be consumed by any LCP-compatible tool.

7 Documentation Coverage

Analyze documentation completeness to identify missing docstrings:

bash 
# Generate coverage report (JSON)
lcp coverage requests -o coverage.json

# Generate coverage report (Markdown)
lcp coverage requests -o coverage.md --format markdown

# Generate both LCP manifest and coverage in one scan
lcp scan requests -o requests.lcp.json --coverage coverage.json

The coverage report shows statistics by symbol kind and lists all undocumented symbols:

summary

High-level coverage statistics with breakdown by symbol kind.

{
  "summary": {
    "total_symbols": 150,
    "documented": 120,
    "undocumented": 30,
    "coverage_percent": 80.0,
    "by_kind": {
      "function": { "total": 60, "documented": 50, "undocumented": 10 },
      "class": { "total": 25, "documented": 20, "undocumented": 5 }
    }
  }
}

Or use the Python API:

python 
from lcp import generate_coverage

report = generate_coverage("mypackage")
print(f"Coverage: {report.summary.coverage_percent}%")

for symbol in report.undocumented:
    print(f"  Missing: {symbol.module}:{symbol.entity}")
