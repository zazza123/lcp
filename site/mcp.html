The Bridge

What is MCP?

Model Context Protocol (MCP) is an open standard for connecting AI assistants to external data sources and tools.

LCP + MCP = Perfect Match

LCP provides the structured library data. MCP provides the communication protocol. Together, they enable AI agents to access accurate API information on-demand.

  • Real-time API queries
  • Version-accurate signatures
  • Zero hallucinations
  • Works with Claude, GPT, and more

Get Started

Quick Start

Set up the LCP-MCP server in under 5 minutes.

1 Install & Generate LCP

bash 
# Install the SDK
pip install lcp

# Generate LCP for your libraries
lcp scan requests -o requests.lcp.json
lcp scan pandas -o pandas.lcp.json

2 Start the MCP Server

bash 
# Start MCP server for a single library
lcp serve requests.lcp.json

# With a custom name
lcp serve requests.lcp.json --name requests-api

3 Configure Your AI Client

Add the server to your MCP client configuration. Select your client below:

Edit your Claude Desktop configuration file:

json 
{
  "mcpServers": {
    "lcp-requests": {
      "command": "lcp",
      "args": ["serve", "/path/to/requests.lcp.json"]
    }
  }
}

Config file location:
macOS: ~/Library/Application Support/Claude/claude_desktop_config.json
Windows: %APPDATA%\Claude\claude_desktop_config.json

Configure MCP in Claude Code using the CLI or settings file:

bash 
# Add MCP server via CLI
claude mcp add lcp-requests -- lcp serve /path/to/requests.lcp.json

# Or for project-specific config, add to your project's settings.json
claude mcp add lcp-requests --scope project -- lcp serve ./requests.lcp.json

Alternatively, edit ~/.claude/settings.json:

json 
{
  "mcpServers": {
    "lcp-requests": {
      "command": "lcp",
      "args": ["serve", "/path/to/requests.lcp.json"]
    }
  }
}

Configure MCP servers for GitHub Copilot CLI in your settings:

bash 
# Add MCP server to Copilot CLI
copilot mcp add lcp-requests lcp serve /path/to/requests.lcp.json

# List configured servers
copilot mcp list

Or edit the config file directly at ~/.copilot/mcp.json:

json 
{
  "servers": {
    "lcp-requests": {
      "command": "lcp",
      "args": ["serve", "/path/to/requests.lcp.json"]
    }
  }
}

Configure MCP servers for OpenAI Codex CLI:

bash 
# Start Codex CLI with MCP server
codex --mcp-server "lcp serve /path/to/requests.lcp.json"

# Or configure in ~/.codex/config.yaml

Add to your ~/.codex/config.yaml:

yaml 
mcp_servers:
  lcp-requests:
    command: lcp
    args:
      - serve
      - /path/to/requests.lcp.json

Configure MCP servers for OpenCode:

bash 
# Add MCP server to OpenCode
opencode config set mcp.servers.lcp-requests.command "lcp"
opencode config set mcp.servers.lcp-requests.args '["serve", "/path/to/requests.lcp.json"]'

Or edit ~/.opencode/config.json:

json 
{
  "mcp": {
    "servers": {
      "lcp-requests": {
        "command": "lcp",
        "args": ["serve", "/path/to/requests.lcp.json"]
      }
    }
  }
}

list_symbols Browse symbols with optional filtering by module or kind (function, class, method). [{ id, kind, summary }, ...]

search_symbols Find symbols by text search across names, summaries, and descriptions. [{ id, kind, summary }, ...]
get_class_members Get all methods and attributes of a class. Essential for understanding class APIs. { methods: [...], attributes: [...] }
get_usage_guide Get strategic guidance on how to efficiently use the LCP manifest. { workflow, tips, mistakes }
explore_return_type Analyze return type of a function and find related classes. { return_type, related_classes }
get_suggestions Get smart suggestions for exploring the library based on your task. { modules, symbols, next_steps }

Programmatic Usage

You can also create and customize the MCP server programmatically:

python 
from lcp.mcp_server import create_server, run_server

# Create a customized server
server = create_server(
    "path/to/manifest.lcp.json",
    name="my-library-api"
)

# Or run directly with defaults
run_server("path/to/manifest.lcp.json")

# You can also load multiple manifests
from lcp import scan

# Generate and serve on-the-fly
lcp_doc = scan("requests")
lcp_doc.to_file("/tmp/requests.lcp.json")
run_server("/tmp/requests.lcp.json")
