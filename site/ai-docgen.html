The Idea

What is AI DocGen?

The LCP coverage command identifies undocumented symbols in your Python packages. AI DocGen takes that report, organizes symbols into a hierarchy, and uses parallel LLM agents to generate high-quality docstrings from the bottom up.

Coverage + AI = Complete Docs

LCP Coverage finds the gaps. AI DocGen fills them. The result is a fully documented codebase with minimal manual effort, using Google-style docstrings compatible with mkdocstrings/mkdocs.

  • Hierarchical bottom-up processing
  • Async parallel LLM agents
  • Context flows up: methods inform classes, classes inform modules
  • Failure propagation with configurable threshold
  • Dry-run mode & token usage tracking
Pipeline

How It Works

1. Coverage
Run lcp coverage to identify undocumented symbols and generate a JSON report.
→
2. Build Hierarchy
Symbols are grouped by module and organized into a tree with levels: leaves, classes, modules.
→
3. Generate Bottom-Up
Async agents process each level in parallel, passing generated context upward to parent levels.
→
4. Write
Docstrings are injected into source files via AST analysis, bottom-up to preserve line offsets.

Get Started

Quick Start

Install the AI extras and start generating docstrings.

1 Install with AI extras

bash 
pip install lcp[ai]

This installs both openai and anthropic SDK packages. You'll need an API key for whichever provider you choose. Set OPENAI_API_KEY or ANTHROPIC_API_KEY as environment variables.

2 Generate a coverage report

bash 
# Analyze your package to find undocumented symbols
lcp coverage mypackage -o coverage.json

3 Run docgen (dry-run first)

bash 
# Preview what would change (no files modified)
lcp docgen coverage.json --provider openai --dry-run

# Generate for real (hierarchical mode is the default)
lcp docgen coverage.json --provider openai --workers 4

# Review the changes
git diff

Advanced

Python API

Use the AI module programmatically for custom workflows and integrations:

python 
from lcp.ai import DocGenAgent, HierarchicalConfig, OpenAIProvider

# Choose a provider
provider = OpenAIProvider(model="gpt-4o")

# Configure hierarchical mode (default)
config = HierarchicalConfig(
    kinds=["class", "function", "method"],
    description="A REST API framework",
    max_workers=4,
    failure_threshold=0.5,
    dry_run=False,
)

# Create and run the agent
agent = DocGenAgent(provider=provider, config=config)
result = agent.run_sync("coverage.json")

# Inspect results
print(f"Processed: {result.symbols_processed}")
print(f"Updated: {result.symbols_updated}")
print(f"Failed: {result.symbols_failed}")
print(f"Tokens: {result.total_usage.input_tokens} in / "
      f"{result.total_usage.output_tokens} out")

for r in result.results:
    print(f"  {r.symbol_id}: {r.status}")
    if r.docstring:
        print(f"    {r.docstring[:80]}...")
Benefits

Why AI DocGen?

Hierarchical Context

Methods are documented first, their docstrings inform class documentation, and class summaries inform module docs. Each level gets precisely the right context.

Async Parallel Agents

Symbols at each level are processed concurrently with configurable workers. True async I/O via asyncio means no wasted time waiting for LLM responses.

Safe & Reversible

Use --dry-run to preview all changes. Then run for real and review with git diff before committing.

Failure Propagation

If too many child symbols fail, the parent is automatically skipped. No wasted LLM calls on symbols lacking sufficient context. Configurable via --failure-threshold.
