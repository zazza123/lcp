The Idea

What is AI DocGen?

The LCP coverage command identifies undocumented symbols in your Python packages. AI DocGen takes that report and uses an LLM to generate high-quality docstrings, writing them directly into your source code.

Coverage + AI = Complete Docs

LCP Coverage finds the gaps. AI DocGen fills them. The result is a fully documented codebase with minimal manual effort, using Google-style docstrings compatible with mkdocstrings/mkdocs.

  • Google-style docstrings
  • Reads actual source code for context
  • Filter by symbol kind
  • Dry-run mode for safe previews
  • Token usage tracking
Pipeline

How It Works

1. Coverage
Run lcp coverage to identify undocumented symbols and generate a JSON report.
→
2. Read Context
For each symbol, the agent reads the source code to provide context to the LLM.
→
3. Generate
The LLM generates a Google-style docstring based on the source code and signature.
→
4. Write
Docstrings are injected into the source files via AST analysis, preserving formatting.

Get Started

Quick Start

Install the AI extras and start generating docstrings.

1 Install with AI extras

bash 
pip install lcp[ai]

This installs both openai and anthropic SDK packages. You'll need an API key for whichever provider you choose. Set OPENAI_API_KEY or ANTHROPIC_API_KEY as environment variables.

2 Generate a coverage report

bash 
# Analyze your package to find undocumented symbols
lcp coverage mypackage -o coverage.json

3 Run docgen (dry-run first)

bash 
# Preview what would change (no files modified)
lcp docgen coverage.json --provider openai --dry-run

# Generate for real
lcp docgen coverage.json --provider openai

# Review the changes
git diff

Advanced

Python API

Use the AI module programmatically for custom workflows and integrations:

python 
from lcp.ai import DocGenAgent, DocGenConfig, OpenAIProvider

# Choose a provider
provider = OpenAIProvider(model="gpt-4o")

# Configure the run
config = DocGenConfig(
    kinds=["class", "function", "method"],
    description="A REST API framework",
    dry_run=False,
)

# Create and run the agent
agent = DocGenAgent(provider=provider, config=config)
result = agent.run("coverage.json")

# Inspect results
print(f"Processed: {result.symbols_processed}")
print(f"Updated: {result.symbols_updated}")
print(f"Failed: {result.symbols_failed}")
print(f"Tokens: {result.total_usage.input_tokens} in / "
      f"{result.total_usage.output_tokens} out")

for r in result.results:
    print(f"  {r.symbol_id}: {r.status}")
    if r.docstring:
        print(f"    {r.docstring[:80]}...")
Benefits

Why AI DocGen?

Context-Aware

The agent reads the actual source code of each symbol, so the generated docstrings accurately describe behavior, parameters, and return values.

Safe & Reversible

Use --dry-run to preview all changes. Then run for real and review with git diff before committing.

Selective Generation

Filter by kind (--kinds class,function) to focus on the symbols that matter most. Skip constants, attributes, or modules.

Cost Transparent

Every run reports total token usage (input, cache, output, reasoning) so you know exactly what each generation costs.
