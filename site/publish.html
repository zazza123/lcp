Publish to the Registry

Share LCP manifests with the community. One command scans, validates, and submits your manifest via GitHub Pull Request.

Workflow

How Publishing Works

1
Scan
Package is scanned and validated against the LCP schema.
→
2
Fork
Your GitHub fork of the registry is created automatically.
→
3
Upload
Gzip-compressed manifest uploaded to the sharded registry path.
→
4
Pull Request
A PR is opened with structured metadata and labels.
Get Started

Quick Start

Step 1: Prerequisites

bash 
# Install the SDK
pip install lcp

# Set your GitHub token (needs repo or public_repo scope)
export LCP_GITHUB_TOKEN=ghp_your_token_here

Step 2: Publish

bash 
# Publish any installed Python package
lcp publish requests

# Preview without creating a PR
lcp publish requests --dry-run

# Use an existing manifest file
lcp publish requests --file requests.lcp.json

Step 3: Verify

Your PR will appear on the registry repository. Once merged, the manifest becomes available to all lcp serve-all --registry users.

Architecture

Registry Structure

Manifests are stored in a sharded directory structure for fast lookups and minimal merge conflicts. Files are gzip-compressed for ~70% size reduction.

Path Template
manifests/{language}/{first_letter}/{name}/{version}.lcp.json.gz
Example
manifests/python/r/requests/2.31.0.lcp.json.gz
text 
lcp-registry/
├── manifests/
│   └── python/
│       ├── f/
│       │   └── flask/
│       │       └── 3.0.0.lcp.json.gz
│       ├── n/
│       │   └── numpy/
│       │       ├── 1.26.0.lcp.json.gz
│       │       └── 2.0.0.lcp.json.gz
│       └── r/
│           └── requests/
│               └── 2.31.0.lcp.json.gz
└── README.md
Advanced

Programmatic Usage

Use the Python API for custom publishing workflows, CI/CD pipelines, or batch operations.

python 
from lcp import scan, publish_manifest

# Scan and publish
doc = scan("requests")
result = publish_manifest(
    doc,
    github_token="ghp_...",
    registry_repo="zazza123/lcp-registry"
)

print(f"PR created: {result.pr_url}")
print(f"Manifest path: {result.manifest_path}")
