Claude Code Integration

LCP Plugin for
Claude Code

Install once, and Claude Code automatically starts an LCP MCP server. Any pip-installed Python library becomes available on demand — no configuration needed.

Features

Everything Included

Auto MCP Server

lcp serve-all starts automatically when Claude Code opens a session. No manual server setup needed.

On-Demand Resolution

Call resolve_library("requests") and the library is scanned, cached, and ready to explore.

Guided Skills

Two built-in skills (lcp-universal, lcp-usage) teach Claude the optimal API exploration workflow.

Quick Commands

/lcp:resolve and /lcp:scan shortcuts for common operations.

Library Explorer

Dedicated read-only subagent for deep API research without consuming parent context.

Registry Support

Configure private or team registries via userConfig.registries for pre-built manifests.

Get Started

Installation

Step 1: Prerequisites

Ensure lcp is installed and available on your PATH.

bash 
# Ensure lcp is installed and on your PATH
pip install lcp
lcp --version

Step 2: Install the Plugin

Install the plugin directly from the repository or a local clone.

bash 
# Install from the repository
claude plugin install plugin/lcp

# Or from a local clone
git clone https://github.com/zazza123/lcp.git
claude plugin install lcp/plugin/lcp

Step 3: Verify

Open a new Claude Code session. You should see the LCP MCP server start automatically. Try asking Claude about any Python library.

Architecture

How the Plugin Works

When Claude Code starts a session, the plugin automatically boots an LCP MCP server that can resolve any pip-installed library on demand.

Startup flow

1
Claude Code session starts
↓
2
Plugin reads .mcp.json → starts bin/serve.sh
↓
3
serve.sh validates lcp on PATH
↓
4
lcp serve-all starts (with optional --registry)
↓
MCP server ready — Claude can resolve any library

Teams

Private Registry Support

Teams can host private registries with pre-built manifests for internal packages. Configure during plugin install or update later.

bash 
# Configure a private registry URL
claude plugin config lcp registries "https://your-registry.example.com"

# Multiple registries (first match wins)
claude plugin config lcp registries "https://internal.example.com,https://public.example.com"

The wrapper script serve.sh reads the CLAUDE_PLUGIN_CONFIG_REGISTRIES environment variable and passes the first URL to lcp serve-all --registry. This allows teams to share pre-built manifests for internal or proprietary packages.

Help

Troubleshooting

"lcp not found" warning

Install with pip install lcp and ensure it's on your PATH. The SessionStart hook checks this automatically and will warn you if lcp is missing.

MCP server not starting

Check that lcp serve-all works from your terminal. The wrapper script at bin/serve.sh needs execute permission — run chmod +x bin/serve.sh if needed.

Library not resolving

Ensure the package is pip-installed in the same Python environment. Try lcp scan <package> manually to verify the library can be introspected.
